## 1. Суть проекта

Персональный сайт нового формата для **Анатолия Веретенникова** — на пересечении двух экспертиз:
- AI-автоматизация и бизнес-приложения для крупных организаций
- Корпоративные фильмы, презентационные ролики, визуальный контент

**Ключевая идея позиционирования:**  
Большинство умеют либо строить системы, либо рассказывать истории. Редкий человек делает и то, и другое. Это незанятый рынок.

### Двухуровневая модель
```
Анатолий Веретенников (личное имя, репутация, гарантия)
        │
        └── [ Название студии ] (услуги, кейсы, команда)
```

### Название студии
**Veretennikov Studio** — личное имя как бренд, без изобретения нового.

## 3. Философия сайта

**Дизайн:** Минимализм, за которым виден профессиональный подход. "Делаем сложное просто" — принцип отражается в самом сайте.

**Тон:** Думающий практик с позицией. Пишем как человек, который много сделал и знает зачем. Никакого buzzword-bingo.

**Анимации:** Lottie/JSON-анимации — только там, где оправданы. Анимация должна объяснять или подтверждать действие, не украшать.

## 6. Клиентский портал проекта

### Концепция
Заказчик получает приватную ссылку на профиль своего проекта. Всё в одном месте, без Telegram и Google Drive вперемешку.

### Что видит клиент
- Этапы проекта с прогрессом (Бриф → Сценарий → Производство → Правки → Сдача)
- Файлы (сценарий, ТЗ, превью, исходники)
- Правки — написать, отследить статус, согласовать
- Референсы — добавить ссылки или файлы
- Сообщения — хронология диалога
- Документы — бриф, договор, смета

### Что видит студия (админ-панель)
- Все проекты и их статусы в одном месте
- Непрочитанные правки и уведомления → в Telegram
- Создание проекта → система генерирует закрытую ссылку
- Шаблоны проектов по типу (фильм, AI-система, презентация)
- Кнопка "Опубликовать в портфолио" для готового проекта
- Аналитика: сколько правок, на каком этапе задержки, история решений

### Версионирование файлов
```
Правка написана → файл загружен (версия N) → правка закрыта → согласовано
```
Это создаёт чистую цепочку и аналитику.

### Авторизация клиента
Magic Link — клиент получает письмо, нажимает, он уже внутри. Без паролей.

## 7. Стратегия анимаций (Lottie)

### Где ДА
| Место | Что | Зачем |
|-------|-----|-------|
| Hero сайта | Схема → чистая форма | Доказывает позиционирование "сложное просто" |
| Этап выполнен | Галочка прорисовывается | Обратная связь на действие |
| Загрузка файла | Документ "летит" в список | Подтверждение действия |
| Empty states | Тихая иллюстрация с жизнью | Приглашение, не баг |
| 404 страница | Что-то с характером | Запоминается, делятся |
| Лоадер (если нужен) | Логотип "собирается" | Бренд в момент ожидания |

### Где НЕТ
- Фоновые частицы, сетки, потоки данных — клише
- Иконки услуг "для красоты" — тормозит, не несёт смысл
- Любые декоративные анимации без смысловой нагрузки

### Технические правила
- Максимум 30–50 кб на один Lottie-файл
- Только векторные пути, без растра внутри JSON
- `autoplay` только когда элемент в viewport (`useInView`)
- `@media (prefers-reduced-motion)` — обязательно

## 8. Технический стек

### Фронтенд
```
Next.js 15 (App Router)   — SSR + SSG + API в одном проекте
TypeScript                — типобезопасность
Tailwind CSS              — дизайн-система
Lottie-react              — JSON-анимации
Framer Motion             — переходы страниц, микроанимации UI
```

### Бэкенд / API
```
Next.js Route Handlers    — API внутри проекта, без отдельного сервера
Prisma ORM                — схема как источник правды, автомиграции
NextAuth v5               — Magic Link авторизация для клиентов
Resend + React Email      — уведомления о правках на почту
```

### База данных и хранилище
```
PostgreSQL (Neon)          — serverless, бранчинг БД как в Git
Cloudflare R2              — файлы проектов, S3-совместимый, без платы за трафик
Redis (Upstash)            — сессии, rate limiting, real-time уведомления
```

### Инфраструктура
```
Vercel                    — деплой, edge функции, превью PR
Cloudflare                — CDN, субдомены show.studio.ru, project.studio.ru
GitHub Actions            — CI/CD, автодеплой на push в main
```

## 9. Схема базы данных (черновик)

```prisma
model Project {
  id          String    @id @default(cuid())
  slug        String    @unique          // для портала и show-страниц
  title       String
  clientName  String
  type        ProjectType               // FILM | AI_SYSTEM | PRESENTATION | SYNTHESIS
  status      ProjectStatus             // DRAFT | ACTIVE | REVIEW | DONE
  isPublic    Boolean   @default(false) // опубликован в портфолио
  accessToken String    @unique         // магическая ссылка для клиента
  createdAt   DateTime  @default(now())
  stages      Stage[]
  files       File[]
  revisions   Revision[]
  messages    Message[]
  references  Reference[]
}

model Stage {
  id        String      @id @default(cuid())
  projectId String
  name      String
  order     Int
  status    StageStatus // PENDING | IN_PROGRESS | DONE
  project   Project     @relation(fields: [projectId], references: [id])
}

model Revision {
  id          String         @id @default(cuid())
  projectId   String
  text        String
  fromClient  Boolean        @default(true)
  status      RevisionStatus // NEW | IN_PROGRESS | DONE | APPROVED
  createdAt   DateTime       @default(now())
  resolvedAt  DateTime?
  fileVersion File?          @relation(fields: [fileVersionId], references: [id])
  fileVersionId String?
  project     Project        @relation(fields: [projectId], references: [id])
}

model File {
  id         String   @id @default(cuid())
  projectId  String
  name       String
  url        String   // Cloudflare R2
  type       String   // mime type
  version    Int      @default(1)
  uploadedAt DateTime @default(now())
  project    Project  @relation(fields: [projectId], references: [id])
  revisions  Revision[]
}
```

## 12. Ключевые принципы, которые нельзя нарушать

1. **Сайт — живое демо экспертизы.** Если показываем AI и современный стек — сам сайт должен быть на современном стеке.
2. **Минимализм со смыслом.** Каждый элемент либо несёт информацию, либо подтверждает действие. Декора нет.
3. **Анимация объясняет, не украшает.** Если анимацию убрать — пользователь чего-то лишается. Иначе — не нужна.
4. **Личное имя = гарантия, студия = механизм.** Имя в нужных местах, "мы" в остальных.
5. **Синтез — главное УТП.** Кейсы "система + история" выводим на первый план.
6. **Портал — конкурентное преимущество.** Клиент не знает Telegram-хаоса — только профессиональный процесс.
