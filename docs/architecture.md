# Architecture

Ключевые технические решения и их обоснование.

## Хостинг: Timeweb Cloud вместо Vercel

**Решение:** VPS на Timeweb Cloud (Россия).

**Почему:** основные заказчики студии в России. Timeweb предоставляет
весь нужный стек — управляемый PostgreSQL, S3-хранилище — в одном месте
и в российской юрисдикции. Vercel + Neon тоже был бы рабочим вариантом
(Frankfurt ~40–80 мс для России), но Timeweb закрывает всё локально.

**Компромисс:** нет нативного Next.js деплоя как на Vercel, CI/CD
настраивается через GitHub Actions + SSH вручную.

---

## Анимации: CSS/SVG вместо Lottie

**Решение:** `@keyframes` + SVG `stroke-dasharray` для hero-диаграммы,
Framer Motion для переходов и scroll-reveal.

**Почему:** бриф разрешал CSS-симуляции вместо Lottie. Чистый CSS/SVG
не требует дополнительных зависимостей, работает без JS-рантайма,
полностью поддерживает `prefers-reduced-motion`.

**Где Framer Motion:** только там где CSS не справляется —
`PageTransition` (переходы между страницами) и `FadeIn` (scroll-reveal
для секций). Не для декора.

## Данные кейсов: статика → Prisma

**Сейчас:** кейсы хранятся в `src/data/cases.ts` как TypeScript-массив.
Страницы `/show/[slug]` генерируются статически при сборке (SSG).

**Потом:** когда появится клиентский портал, кейсы переедут в PostgreSQL.
Страницы показа станут ISR (revalidate по событию публикации).

**Почему не сразу в БД:** нет смысла поднимать Prisma пока нет портала.
Статика проще, быстрее и не требует env-переменных для публичной части.

## Три зоны сайта

```
┌─────────────────────────────────────────────┐
│  ПУБЛИЧНАЯ ЗОНА       /                     │
│  app/(public)/        SSG, без auth          │
├─────────────────────────────────────────────┤
│  СТРАНИЦЫ ПОКАЗА      /show/[slug]          │
│  app/show/            SSG, без auth          │
│  Шарится с клиентами напрямую               │
├─────────────────────────────────────────────┤
│  КЛИЕНТСКИЙ ПОРТАЛ    /project/[id]         │
│  app/project/         Magic Link auth        │
│  ПЛАНИРУЕТСЯ                                │
└─────────────────────────────────────────────┘
```

Зоны физически разделены в файловой структуре Next.js App Router.
`(public)` — route group, не влияет на URL.

## Бренд: личное имя + студия

**Правило:** Анатолий Веретенников = гарант и продюсер. Студия говорит
«мы» везде кроме: футер («Основана Анатолием Веретенниковым»), страница
«О студии», контакты.

**Почему важно в коде:** любые тексты, написанные от первого лица «я»
в публичных компонентах — ошибка позиционирования.

---

## Навигация: задачи, не инструменты

Структура навигации строится вокруг задач клиента:

```
Задачи → решения   /services    (не "AI" и не "Видео" отдельно)
Кейсы              /cases
О подходе          /about
Манифест           /manifesto
Связаться          /contact
```

Нет разделов «Видеопродакшн» и «AI-разработка» — это инструменты студии,
а не задачи клиента.

## Планируемый бэкенд

```
Prisma ORM         — схема как источник правды
Neon / Timeweb PG  — PostgreSQL
NextAuth v5        — Magic Link для клиентов
Resend             — email-уведомления о правках
Timeweb S3         — файлы проектов (S3-совместимый)
Redis (Upstash)    — сессии, rate limiting
```
